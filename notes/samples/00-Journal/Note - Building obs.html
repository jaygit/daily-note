Building obs — CLI Obsidian Companion

Saved: 2025-12-13   •   tags: journal, blog, obs

I wanted a lightweight, platform-agnostic way to create and manage notes from the shell — a command-line companion for Obsidian vaults that favors openness and composability. Because Obsidian stores plain Markdown in a folder, it’s naturally extensible: any CLI tool can create, edit, and link notes. This project, packaged as the obs helper scripts, grew from that idea.

Motivation

  • Why CLI-first: Quick capture, scripted workflows, and easy automation — the terminal remains the fastest way for me to create structured notes while I’m already in a shell environment.
  • Why Obsidian-friendly: Obsidian’s vault is just a directory of Markdown; staying compatible keeps the data portable and editable by many tools. This openness allowed me to iterate on tooling without locking content into a closed format.

Design Goals

  • Small, composable scripts in scripts/ (e.g., create_note.sh, search.sh, gitnotes.sh).
  • Platform-agnostic installer using XDG paths, installs only the scripts/ tree and a tiny obs shim.
  • Persist runtime settings into scripts/.env so installed scripts pick up VAULT_DIR, GIT_REMOTE_*, and VERSION.
  • Pragmatic fallbacks: prefer rg, fzf, bat but support grep, cat, and test stubs.
  • Testable CI with deterministic stubs and artifact logging.

Step-by-step journey

  1. Basic capture and templating: create_note.sh generates Markdown with category-aware frontmatter, filename sanitation, and optional editor integration.
  2. Helpers & shared utilities: lib.sh provides prompts, sanitizers and load_dotenv().
  3. Interactive search and previews: search.sh wires rg + fzf + bat into a fuzzy-search UI with diary mode and fallbacks.
  4. Git integration: gitnotes.sh manages vault-scoped commits and remote interactions.
  5. Tests: shell tests under tests/ with deterministic stubs for CI.
  6. Installer: XDG-aware install.sh persists VERSION, creates an obs shim and supports --uninstall.
  7. CI hardening & releases: workflow fixes, host-resolution fallbacks, and release/tag discovery from RELEASE_NOTES.

Future directions

  • Plugin hooks for post-create actions (auto-tagging, external indexing).
  • Richer frontmatter templates and community templates.
  • Smarter merge/conflict helpers for collaboration.
  • Graph/metadata export (JSON/SQLite) for local indices.
  • Minimal GUI bridges wrapping the CLI for mouse-driven workflows.

How to reuse

  • Edit scripts/.env after installing to change VAULT_DIR or remote config.
  • Override FZF_CMD for deterministic tests.
